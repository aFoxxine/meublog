O objetivo desta matéria é apresentar os fundamentos do HTML (Hypertext Markup Language) com ênfase na semântica do uso de elementos e nos benefícios do emprego de padrões de codificação, assim como demonstrar uso do CSS (Cascading Style Sheets) para separar conteúdo e apresentação, de modo a aumentar velocidade do desenvolvimento e as habilidades de design. Abaixo veja um resumo da matéria e algumas atividades feitas por mim.

A Bagunça

A Web cresce comercialmente e vem a necessidade de publicação de conteúdo diagramado, algo para o qual o HTML não possuía recursos. Há, então, a adaptação da linguagem de forma equivocada, com uma preocupação unicamente visual, esquecendo da qualidade do código.

Aproximadamente entre 1995 e 1999 ocorre a chamada “Guerra dos Browsers”, em que a Netscape (e seu Browsers de mesmo nome) e a Microsoft (com o seu Internet Explorer) disputavam o mercado de navegadores. Além de não darem suporte aos padrões do recém criados pela World Wide Web Consortium (W3C), ainda criavam seus próprios padrões, aumentando a bagunça.

Os Padrões Web

Padrões Web ou Web Standards são um conjunto de normas, diretrizes, recomendações, notas, artigos, tutoriais e afins de caráter técnico, produzidos pelo W3C e destinados a orientar fabricantes, desenvolvedores e projetistas para o uso de práticas que possibilitem a criação de uma Web acessível a todos, independentemente dos dispositivos usados ou de suas necessidades especiais. Alguns exemplos: HTML5, XHTML, HTML, CSS, DOM, XML.

O que é W3C?

W3C (World Wide Web Consortium) é um consórcio de empresas de tecnologia. Fundado por Tim Berners-Lee em 1994 para levar a Web ao seu potencial máximo. Desenvolve Recomendações abertas, até agora mais de 80 padrões.

Web dividida em três camadas:

Devem ser desenvolvidas de forma independente, porém uma complementa a outra.

HTML5

Usa-se o HTML5 para a estrutura e conteúdo do site.

CSS

Usa-se o CSS para a apresentação, a parte de design do site.

JavaScript

Usa-se o JavaScript para definir o comportamento do site, por exemplo: caixas de avisos, uma transição de cor, etc.

Vantagens da adoção dos Padrões

A organização feita nessas três camadas e outros padrões traz muitas vantagens, como: carregamento mais rápido, melhor consistência visual, redesign mais barato e eficiente, melhores resultados nos mecanismos de buscas, maior acessibilidade e interoperabilidade.

Um Código Semântico

Semântica refere-se ao estudo do significado. Quando utilizamos cada marcação para o que ela realmente foi criada, estamos construindo um “Código Semântico”, podemos parar de pensar em “isso vem aqui, isso vai ali...” e pensar na “Estrutura da Informação” e criar uma “Marcação com Significado”.
